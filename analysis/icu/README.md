# analysis/icu

Go port of Apache Lucene 10.4.0's `lucene/analysis/icu` module. Provides the
ICU-flavoured analysis surface: collation, normalisation, transliteration, and
the UAX#29 segmentation pipeline.

## ICU4J binary break-iterator dictionaries (`.brk`)

Apache Lucene's `analysis/icu` module ships two compiled
RuleBasedBreakIterator (RBBI) blobs sourced from the ICU4J distribution:

- `Default.brk` ﻗ UAX#29 word segmentation with dictionary-based handling
  for Han, Hiragana, Katakana, Hangul, Thai, Lao, and Khmer.
- `MyanmarSyllable.brk` ﻗ Myanmar syllable segmentation.

Both files are generated by the upstream ICU compilation pipeline against the
ICU4C source tree (see Lucene's
`org.apache.lucene.analysis.icu.RBBIRuleCompiler`). They are not derivable
from the Gocene source tree alone, so Gocene **bundles the upstream blobs** as
embedded production resources under `segmentation/` (`Default.brk`,
`MyanmarSyllable.brk`), byte-identical to the resources Apache Lucene 10.4.0
ships (verified by `TestEmbeddedBRKByteIdentity`). They are exposed via
`segmentation.LoadEmbeddedBRK(name)`; see the NOTICE file for the ICU license.
The copies in `segmentation/testdata/` remain for parser-validation tests.

Note: the blobs are bundled, parseable, and now **executed** at runtime by the
RBBI forward state-machine engine (`segmentation.RBBIBreakIterator`): Myanmar
syllable and CJK per-ideograph segmentation are dictionary/rule driven and
byte-identical to ICU/Lucene (all `TestMyanmarSyllable` cases pass). The
Thai/Lao/Khmer dictionary-refinement pass and reverse iteration remain (tracked
as rmp #4791); those scripts use the UAX#29 rule fallback for now.

### Loader API

`segmentation.LoadBRKDictionary(io.Reader)` and
`segmentation.ParseBRKDictionary([]byte)` accept the same binary format that
ICU4C `udata_open` and ICU4J
`RuleBasedBreakIterator.getInstanceFromCompiledRules(InputStream)` consume.
The loader validates the `UDataInfo` header (magic `0x27da`, dataFormat
`"Brk "`, endianness, charset family) and the `RBBIDataHeader` (magic
`0xb1a0`, length and category-count fields) before returning a
`BRKDictionary` value.

When no blob is supplied:

```go
dict, err := segmentation.LoadBRKDictionary(nil)
if errors.Is(err, segmentation.ErrNoDictionary) {
    // fall back to rules-based segmentation
}
```

Callers can branch on `errors.Is(err, segmentation.ErrNoDictionary)` to detect
the "no dictionary supplied" path and fall back to the rules-based
`goWordBreakIterator` (the default segmentation path today).

### RBBI execution engine

Gocene executes the compiled RBBI rules with a forward state-machine engine.
`BRKDictionary.AsBreakIterator()` returns a real `RBBIBreakIterator` (a
CodePointTrie reader plus the state/status table interpreter ﻗ ports of ICU4J
`CodePointTrie` / `RBBIDataWrapper` / `RuleBasedBreakIterator.handleNext`), and
`BRKDictionary.HasDictionaryExecution()` reports `true`.

The practical consequence: Myanmar syllable segmentation (`MyanmarSyllable.brk`)
and CJK per-ideograph segmentation (`IDEOGRAPHIC`) are driven by the compiled
rules and are byte-identical to ICU/Lucene. The remaining work ﻗ Thai/Lao/Khmer
dictionary-refinement word breaking and reverse iteration ﻗ is tracked as
rmp #4791; those scripts use the UAX#29 rule fallback until it lands.

### Bundling a `.brk` blob

To attach a dictionary at runtime, read the blob and pass it to the loader:

```go
blob, err := os.ReadFile("path/to/Default.brk")
if err != nil { /* ... */ }
dict, err := segmentation.ParseBRKDictionary(blob)
if err != nil { /* ... */ }
_ = dict // pass to a future config that consumes BRKDictionary
```

The byte format is the ICU `udata` envelope (20-byte `UDataInfo` plus a
padded header bringing the total to 32 bytes) followed by the
`RBBIDataHeader` and its trie/state tables. Bytes are little-endian on
disk. The Lucene 10.4.0 source ships authoritative copies at
`lucene/analysis/icu/src/resources/org/apache/lucene/analysis/icu/segmentation/Default.brk`
and `.../MyanmarSyllable.brk`.

### Deferred work

The `.brk` blobs are bundled (with ICU BSD-style notices in `NOTICE`) and the
RBBI forward execution engine (state-table interpreter, CodePointTrie lookup,
status-tag emission) is implemented and drives segmentation. The remaining
parity work, tracked as rmp #4791, is:

1. **Thai/Lao/Khmer dictionary-refinement pass** ﻗ the dictionary lookup that
   refines rule-machine runs into words for those scripts.
2. **Reverse iteration** and the combined-CJK word iterator.
